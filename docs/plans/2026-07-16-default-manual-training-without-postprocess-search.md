# 默认人工特征训练与关闭后处理搜参实施计划

## 目标

保证默认主流程先生成完整特征排序 CSV 并暂停；用户人工选择特征后，恢复流程严格使用 CSV 中选中特征的名称、顺序和数量完成模型搜参、hard-negative 优化、评估与部署导出。默认不执行后处理参数搜索。

## 实施步骤

1. 为默认首阶段 dry-run 增加验收测试：只包含 `s01`–`s04`，明确暂停，不包含后处理搜索命令。
2. 为人工 CSV 恢复 dry-run 增加验收测试：包含模型搜参、最大深度 2–5、hard-negative、评估和部署导出，不包含特征数量搜索与后处理搜索。
3. 核对并在必要时修正 `s08_run_pipeline.py` 的默认参数、阶段门控、日志和帮助文本。
4. 核对 `s05_train_model.py`、人工 CSV 读取和部署导出链路，确保选中特征名称、顺序、数量一致。
5. 更新 README，把两阶段人工流程放在首要位置，并明确后处理搜索仅通过显式参数启用。
6. 运行定向测试、全量测试、Python 编译、错误级静态检查与差异检查。

## 验收标准

- 默认命令运行到 `s04` 后暂停。
- 人工恢复命令不搜索特征数量，且默认执行充分模型搜参与 hard-negative 优化。
- 默认不运行 `s06_opt`、窗口缓存导出和 `s07_post`。
- 部署模型与独立特征脚本严格使用人工 CSV 的选中特征顺序。
- README、代码注释、CLI 帮助和实际默认行为一致。
- 全量验证无错误。
