# 删除 Stage1、统一为纯 XGBoost 的实施计划

## 目标

- 从活动流程中彻底删除 Stage1（IR DC、AC/DC 阈值、门控与融合）。
- 第二阶段特征生成、模型训练、窗口/样本评估和部署产物只依赖 XGBoost。
- 默认最终统计不叠加 EMA、投票或状态机；这些后处理仅保留为可选独立分析。

## 实施步骤

1. 增加回归测试，复现 Stage1 全未通过时 TP/FP 被清零的问题，并约束活动入口不再依赖 Stage1。
2. 删除 Stage1 阈值脚本及特征池中的 Stage1 派生特征，更新特征池版本与说明。
3. 重构部署评估：移除 Stage1 流式门控、融合结果和配置导出，主结果改为原始 XGBoost 概率加模型阈值。
4. 从总流水线、验收脚本、README 和部署文档中移除 Stage1 步骤及并行/融合叙述。
5. 运行定向测试、全量测试、语法检查和入口帮助检查，确认运行链路一致。

## 验收标准

- 不再生成或读取 `stage1_threshold.json`、`stage1_config.json`。
- 最终混淆矩阵不再受 IR DC/AC/DC 条件影响。
- `s08_run_pipeline.py` 不再调度 `s02_ir_dc_threshold.py`。
- 部署包只需要独立特征提取脚本、XGBoost 模型及其特征/阈值元数据。
- 全量自动化测试通过。
