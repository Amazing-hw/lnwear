# 并行训练与确定性特征提取设计

## 目标

在不改变人工特征集合、特征数量、Stage1/Stage2 业务语义和模型选择规则的前提下，补齐 s05 模型搜索并行能力，并保证 s03 多进程特征提取的输出顺序可复现。

## 方案选择

s05 采用外层候选线程池：多个独立 XGBoost 候选并行训练，每个候选默认 `n_jobs=1`。线程共享只读 NumPy 特征矩阵，避免 Windows 多进程为每个 worker 复制完整训练数据。候选结果写入预分配索引位置，选择结果不依赖完成先后顺序。若高级用户显式提高 `WL_INNER_N_JOBS`，需要同步降低外层 worker，避免两层并行相乘。

未采用两种替代方案：XGBoost 单模型扩大 `n_jobs` 容易与候选搜索嵌套抢占；候选级多进程隔离更强，但 Windows 下矩阵复制、模型序列化和峰值内存更高。

## 数据流

1. s08 的全局 `--n_workers` 传给 s05 的 `--model_search_n_workers`。
2. s05 Stage A、Stage B 和 single-split 搜索通过统一的有序线程映射执行候选任务。
3. 每个任务只返回指标、参数和节点数；父线程按照输入索引重组记录。
4. 选定最佳候选后，仅在主线程重训最终模型，避免同时保留全部候选模型。
5. s03 多进程 worker 结果先写入 `rows_by_sample[index]`，全部完成后按原始样本顺序展开。

## 并发与错误处理

- `model_search_n_workers<=1` 或只有一个任务时走原串行路径。
- `WL_FORCE_SERIAL=1` 同时关闭 s05 外层并行。
- XGBoost、BLAS、OpenMP 内层线程默认保持 1，避免 `worker × inner thread` 过度订阅。
- 任一模型候选异常直接传播并终止该次搜索；不允许静默丢失候选后继续给出不完整最优结果。
- s03 单个样本失败仍沿用现有警告和空结果语义，但其他样本的相对顺序保持确定。

## 兼容性

- s05 直接调用者未提供 `model_search_n_workers` 时默认 1，保持旧行为。
- manual 模式只固定特征集合和数量，仍可并行搜索模型参数。
- 模型评分、阈值、节点上限、默认参数保护和 hard-negative 接受条件不改变。

## 验收

- 测试先证明并行任务乱序完成时，返回结果仍与输入顺序一致。
- 测试证明 `WL_FORCE_SERIAL` 能强制 s05 使用单 worker。
- 测试证明 s03 在 worker 逆序完成时仍按输入样本顺序输出。
- dry-run 显示 s05 收到与主流程一致的 worker 数。
- 运行模型搜索、预窗口 H5、端到端保护、人工 CSV 和全量测试。
