# Interpretable Stage2 Feature Pool Design

> Historical design record. The active implementation is v8 with 126 governed
> candidates, CSV-only manual selection and fixed-position three-zone features.
> The commercial comparison runtime was removed; its eight-feature mapping remains.

Date: 2026-07-10
Status: Approved for implementation

## 1. Goal

Replace the current 208-column Stage2 candidate surface with a smaller, explicitly governed feature pool whose members have a traceable signal meaning, stable preprocessing semantics, bounded numerical behavior, and a practical path to C firmware.

The data-driven ranking remains responsible for ordering every eligible candidate. The final training list remains a human decision. This change governs which features are eligible to appear in that ranking.

## 2. Scope

This change covers:

- Stage2 green, ambient, and ACC feature extraction.
- Candidate versus diagnostic-only column policy.
- Feature grouping used by ranking and deployment-cost summaries.
- Formula metadata, preprocessing metadata, and C-port metadata.
- Strict pool-version validation between extraction, ranking, training, and deployment.
- Python golden-vector exports that define the expected C implementation behavior.

This change does not alter Stage1 IR gating or the independent commercial eight-feature AdaBoost comparison in `s09_commercial_compare.py`.

## 3. Feature Governance

Add a single catalog module that is the source of truth for:

- Feature pool version.
- Ordered model-candidate feature names.
- Diagnostic-only fields.
- Removed legacy fields and removal reasons.
- Semantic group, unit, signal source, preprocessing path, formula summary, numerical guard, scale dependency, and C operator requirements.

Every generated model candidate must exist in the catalog. Every catalog candidate must be generated by the shared window extractor. `s04` must reject unversioned or mismatched feature-pool CSV files instead of silently accepting stale columns.

## 4. Removal Policy

### 4.1 Constants and metadata shortcuts

Remove from model candidates:

- `SIG_LEN`, `SIG_SEC`, `G_TOP2_CHANNEL_COUNT`.
- `mode`.
- `TOTAL_INVALID_COUNT`, `PPG_INVALID_COUNT`, `GREEN_INVALID_COUNT`.

`mode` and invalid-value counts remain available as diagnostic columns in the feature-pool CSV. They are never ranked or passed to the model.

### 4.2 Channel identity and direction encodings

Stop generating:

- `G_TOP2_WORST_IDX`, `G_MIN_CHANNEL_ID`, `G_DROPOUT_ANGLE`.
- `ACC_DOM_AXIS`.
- ACC per-axis mean and energy features.
- Individual G1/G2/G3 feature expansions.

These fields encode sensor orientation or channel identity and can become hardware/session shortcuts. Symmetric three-channel statistics and magnitude-based ACC features remain.

### 4.3 Exact aliases and deterministic complements

Keep one canonical feature and remove its aliases:

- Keep `GREEN_AC_DC_RATIO`; remove `G_mean_acdc`.
- Keep `GREEN_DC_MEDIAN`; remove `GREEN_DC` and `G_mean_mean`.
- Keep `GREEN_AUTO_CORR_PEAK`; remove `GREEN_XCORR`.
- Keep `GREEN_FFT_PEAK_MEDIAN_RATIO`; remove `FFT_PEAK_MEDIAN_RATIO`.
- Keep `AMBX_DC_MEDIAN`; remove `AMB_DC` and `Ambient_mean`.
- Keep `AMBX_FFT_PEAK_MEDIAN_RATIO`; remove `AMB_FFT_PEAK_MEDIAN_RATIO`.
- Keep `AMBX_DOM_FREQ`; remove `AMB_DOM_FREQ`.
- Keep `ACC_MAG_MEAN`; remove `ACC_YSUM`.
- Keep `G_TOP2_RANK_STABILITY`; remove `G_TOP2_SWITCH_RATE`.

The commercial comparison keeps its own original eight feature names and formulas inside `s09_commercial_compare.py`.

### 4.4 Redundant spatial and consensus expansions

Remove all G1/G2/G3 individual features and the broad `G_consensus_*` expansion. Keep a compact symmetric set:

- `G_imbalance_mean`, `G_imbalance_p90`, `G_imbalance_iqr`.
- `G_rangeNorm_mean`, `G_rangeNorm_p90`.
- `G_ch_dc_cv`, `G_ch_dc_max_min_ratio`.
- `G_bp_corr_mean`, `G_bp_corr_min`, `G_bp_corr_std`.
- `G_2OF3_AC_SUPPORT`, `G_TOP2_TO_ALL_AC_RATIO`, `G_TOP2_CORR_MIN`.
- `G_WEAK_CHANNEL_GAP`, `G_SPATIAL_STABILITY_SCORE`.
- `G_TOP1_TO_TOP2_AC_RATIO`, `G_TOP2_RANK_STABILITY`.

The spatial-vector family is removed because it is mathematically equivalent or near-equivalent to the normalized three-channel spread family for this three-sensor geometry.

### 4.5 Unreliable formulas

Stop generating:

- `GREEN_SAT_FRAC`: the current formula compares values with 98% of the window maximum, not a hardware saturation limit.
- `ACC_ENERGY_TO_GREEN_AC`: window-length-dependent energy including gravity divided by optical AC has unstable units.
- `ACC_STILL_GREEN_MISMATCH`: it can diverge when ACC derivative MAD approaches zero.
- High-order green/ambient skewness and kurtosis.
- Raw extrema and index-based derivative/peak features.

Keep bounded, robust alternatives such as median, IQR, MAD, normalized ratios, correlations, physiological-band energy ratios, autocorrelation peak, and segment stability.

## 5. Preprocessing Contract

Feature families do not share one undifferentiated preprocessing path.

### 5.1 Quality-before-repair

Compute flatness, spike, clipping, finite-value, and dropout evidence from finite-corrected raw input before burr removal, step suppression, median filtering, moving average, or bandpass filtering. Preprocessing must not hide sensor failures.

### 5.2 Contact and light level

Compute DC, IQR, robust range, spatial balance, and ambient-to-green ratios from a minimally cleaned raw path:

1. Replace non-finite samples with the channel finite median.
2. Apply isolated-spike repair only.
3. Do not apply step suppression.
4. Do not apply smoothing before DC and spatial statistics.

Real wear/remove transitions are valid signal content and must not be flattened.

### 5.3 Pulsatile and periodic evidence

Compute AC, correlation, autocorrelation, and frequency features from:

1. Finite correction.
2. Isolated-spike repair.
3. Median detrending with an odd window of approximately 0.8 seconds.
4. Optional short moving average of approximately 40 ms.

Do not use the existing 65-tap forward-backward FIR on 3-5 second windows. Its effective impulse response and edge behavior are expensive to reproduce in embedded C and can dominate short windows. Median detrending plus bounded smoothing is deterministic, phase-neutral for correlation use, and requires only rolling median and arithmetic operators.

### 5.4 ACC evidence

Use orientation-invariant acceleration magnitude. Separate gravity and motion with a rolling mean or median baseline. Compute motion MAD/RMS/stillness from the residual, not from raw magnitude energy. ACC-to-green coupling uses bounded correlation or log-ratio with denominator floors based on signal scale.

## 6. Numerical Robustness

- Ratios use a scale-aware denominator floor, not a universal `1e-12` when ADC-scale values are involved.
- Correlations require a minimum variance in both inputs and return zero otherwise.
- Autocorrelation and FFT features require minimum window length and in-band energy.
- Segment statistics require at least two valid segments.
- All final candidate values must be finite.
- Quality counters record fallback use but remain diagnostic-only.

## 7. C Engineering Contract

Each catalog entry records:

- Required intermediate signals.
- Primitive operations.
- Buffer requirement in samples.
- Whether an FFT is required.
- Expected numeric range when bounded.
- Recommended C accumulator type.

The deployment package exports:

- `stage2_feature_catalog.json` containing only selected features plus shared preprocessing operators.
- `stage2_c_contract.json` containing operator inventory, buffer sizes, FFT sources, feature order, and pool version.
- Existing Python golden vectors, extended with raw selected feature values before fill/clip and tolerances for C parity.

The Python extractor remains the numerical source of truth. A separate unverified C implementation is not generated automatically. Firmware implements the exported operator contract and must pass the golden vectors.

## 8. Compatibility

Old `feature_pool_*.csv`, ranking files, and model bundles are incompatible with the new pool. `s04`, `s05`, and deployment export must fail with an actionable message instructing the user to rerun from `s03` when the pool version is missing or different.

No silent alias insertion, automatic column deletion, or stale-model fallback is allowed.

## 9. Tests

- The generated candidate set exactly equals the ordered catalog set.
- No candidate is assigned to `other`.
- Constants, diagnostic-only fields, channel identifiers, aliases, and unstable formulas are absent from model candidates.
- Diagnostic CSV metadata remains available for stratified analysis.
- Quality features respond to faults before preprocessing repair.
- DC features retain a true step transition while periodic features remain robust to isolated spikes.
- Candidate features remain finite on constant, sparse, clipped, NaN/Inf, and low-amplitude inputs.
- Training and batch extraction use the same window assembler.
- Stale pool versions are rejected.
- Every selected feature exports formula, preprocessing, and C operator metadata.
- Golden vectors preserve exact feature order and include parity tolerances.

## 10. Acceptance Criteria

- The Stage2 model candidate count is between 80 and 120.
- Every candidate has an explicit physical group and catalog record.
- No `other`, mode, channel index, absolute direction, constant, or diagnostic-only field reaches ranking or training.
- Duplicate extraction paths are removed.
- All focused and full regression tests pass.
