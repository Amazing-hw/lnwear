# 全特征生理意义与可解释性手册设计

状态：用户已确认（2026-07-16）

## 目标

生成根目录文档 `FEATURE_INTERPRETABILITY_GUIDE.md`，覆盖 `stage2_interpretable_v8` 特征池全部 126 项，为人工编辑 `manual_feature_selection.csv` 提供生理、物理、泛化和工程依据。

## 信息来源

- 特征全集与顺序以 `stage2_feature_catalog.model_candidate_names()` 为唯一契约。
- 公式、信号源、特征分组、FFT 和端侧算子信息来自 `stage2_feature_catalog.py`。
- 实际实现细节以 `s03_extract_feature_pool.py` 为准。
- 商用映射、人工选择规则和三光区背景以 README 及现有设计文档为准。
- 文档不把统计相关性写成确定的生理因果；无法给出固定变化方向时明确标注“依赖数据分布，应以分层验证为准”。

## 文档结构

1. 使用说明：如何结合完整排名、漂移、成本和分层错误人工筛选。
2. 术语与信号定义：DC、AC、AC/DC、带通、周期性、三光区、整体均值、逐点中位、top2、环境光和 ACC。
3. 126 项完整索引：按 catalog 顺序列出所属章节，方便与 CSV 对照。
4. 分类正文：
   - 配置与信号质量；
   - 整体绿光与基础统计；
   - top2/逐点中位稳健绿光；
   - 环境光基础统计；
   - 频域、周期性与形态；
   - 三光区一致性、失衡与 pair 共识；
   - 三光区固定位置；
   - 绿光—环境光耦合；
   - ACC 运动；
   - ACC—PPG 耦合；
   - 商用方案兼容特征。
5. 人工选择建议：核心组合、互补组合、风险特征、冗余控制和端侧成本控制。

## 每项特征固定字段

每个特征使用独立三级标题，并包含：

- **定义/公式**：用可读的简化公式描述实际实现。
- **生理与物理意义**：解释它对应接触、血容量脉动、周期性、运动、环境光或空间一致性的哪一部分。
- **佩戴判别预期**：说明佩戴/非佩戴的典型趋势；若不存在稳定单调方向则明确说明。
- **鲁棒性价值**：说明对单边翘起、局部漏光、低灌注、运动、强环境光、通道失效的价值。
- **混淆与泛化风险**：列出设备增益、LED 电流、肤色、姿态、硬件配置、信号幅值和样本构成等风险。
- **工程与人工筛选建议**：标注可解释性等级、相对计算成本、主要冗余对象以及建议保留/条件保留/谨慎选择。

## 分类原则

- 同一特征只在一个主体章节完整解释；索引和商用映射允许交叉引用，不重复制造不同定义。
- `mode` 明确标记为硬件配置元数据，不是生理特征，并标记 device shortcut 风险。
- 固定位置 `GZONE1/2/3` 明确标记方向信息和跨设备映射风险。
- top2 明确说明其依据是脉动能量排序，不等价于直接选择“质量最好”的两区。
- 环境相关性特征不简单解释成“越低越好”；真实佩戴也可能受共同运动或光学串扰影响。
- ACC 特征描述运动/姿态，不直接证明活体；必须与 PPG 证据联合解释。
- 商用 8 特征说明原公式关系和当前三光区适配关系。
