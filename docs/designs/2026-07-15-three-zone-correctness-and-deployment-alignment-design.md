# 三光区正确性与部署一致性修订设计

## 目标与边界

本修订解决三光区审查发现的确定性问题，同时保持用户确认的 `mode1/mode2`
通道映射完全不变。受治理候选数量保持 111，特征池版本升级为
`stage2_interpretable_v7`，使旧排序、人工 CSV 和模型产物不能静默复用旧公式。

商用八特征映射和 `COMM_GREEN_AC`、`COMM_AMB_AC` 候选继续保留；商业方案对比脚本、
CLI 步骤、验收分支、图片和 README 说明全部删除。

## 三光区信号与 top2

`GREEN`、`GMEDIAN`、三种 pair 和环境残差分支保持不变。`GTOP2` 仍是 AC 能量优先的
辅助表示，但选择改为并列感知：计算三种 pair 的 AC 分数和，保留与最大分数在数值容差
内并列的所有 pair，再对这些 pair 的逐点均值求平均。无并列时与原公式完全相同；三路
完全并列时退化为三路均值。top2 相关性使用全部并列最优 pair 相关性的中位数，最弱区
相关性使用全部并列最弱区相关性的中位数。这样不再依赖通道编号。

## 有效主频和单区失效

主频必须同时满足：频带非零、频谱峰中位比达到固定门槛、40–180 bpm 自相关峰达到固定
门槛。无效区域不参与主频 MAD、pair 主频差、相位或频谱 pair 统计，但仍在
`G_ZONE_HR_SUPPORT_RATIO` 的三分母中计为不支持。只有一条有效区域时，相位集中度和 pair
频谱共识返回 0；两条有效区域时保留这两区的共识证据。

`compute_fft_cache` 对严格零能量频带输出 `dom_freq=0`。有效性门槛集中在一个受测试的
辅助函数中，并在 catalog 公式、风险标记和 C 操作符说明中显式声明。

## 部署单一真源

`stage2_feature_catalog.py` 是特征顺序、公式、预处理、数值保护和 C 操作符的唯一真源。
`s08` 的 deploy cookbook 不再维护第二套逐特征公式表；所有 111 个候选必须能生成非空、
非 `[未匹配]` 配方。公共预处理说明改为当前主路径：有限值替换、孤立毛刺修复、0.8 秒
滚动中位数基线、pulse 去趋势和 0.04 秒可选平滑。`s06` 导出的
`feature_formulas.json` 使用同一说明。

部署成本评分直接读取每个特征的 `deployment_cost`，不再用 group 到单一成本的覆盖字典
代替单特征成本。FFT 复用源仍单独统计。

## 文档、测试与验收

README 明确 RMS 是能量而非质量，删除全部商业对比入口和产物。新增测试覆盖：精确 RMS
并列的六种排列、两区有效一区平坦、微弱噪声主频无效、111 个 cookbook 配方完整、部署
预处理与主提取器一致、单特征成本进入子集评分，以及项目中不再存在商业对比活动入口。

验收包括定向红绿测试、完整 pytest、生产文件 py_compile、Pylint errors-only、三种
dry-run、PNG-only 扫描和 `git diff --check`。不创建提交或推送。
