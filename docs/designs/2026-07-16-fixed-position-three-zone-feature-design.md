# 固定位置三光区特征设计

## 背景

不同硬件配置可能提供 3、6 或 9 路绿光。原始通道始终可以按表底固定物理位置稳定映射为区域 1、区域 2、区域 3，且区域顺序在硬件版本和数据读取中保持不变。各区域内部先等权平均，形成共享模型统一使用的三路区域信号。

当前特征池主要使用三光区均值、中位数、top2、2-of-3 和 pair 顺序统计等置换不变证据。这些证据泛化性强，但没有利用稳定的绝对区域位置，可能弱化单边翘起、固定方向漏光和局部接触异常的信息。

## 设计目标

- 本文的历史 mode 映射约束已由同日 H5 元数据驱动设计取代；固定位置特征语义保持不变。
- 不修改商用八特征及现有三光区稳健特征公式。
- 不拆分 3、6、9 通道模型，继续训练和部署一个共享模型。
- 只向受治理特征池增加位置感知候选，不强制选入模型。
- 最终特征及数量继续由完整排序后的人工 CSV 决定。
- 新特征必须有限、可解释、可导出到 C，并尽量避免绝对 ADC 尺度和通道数量捷径。

## 方案

保留现有全部全局和置换不变特征，新增 15 项固定位置候选。每个物理区域分别生成以下五项：

1. `GZONE{1,2,3}_DC_CONTRAST`
   - 定义：`(zone_dc - median(zone_dc)) / (abs(zone_dc) + abs(median(zone_dc)) + guarded_eps)`。
   - 目的：检测固定位置的局部接触、翘起和异常入光，同时降低整体 ADC 尺度影响。
2. `GZONE{1,2,3}_AC_CONTRAST`
   - 定义：`(zone_ac - median(zone_ac)) / (zone_ac + median(zone_ac) + guarded_eps)`。
   - 目的：描述固定位置的相对脉动强弱。
3. `GZONE{1,2,3}_AC_DC_RATIO`
   - 定义：`RMS(zone_pulse) / guarded_abs(median(zone_raw))`。
   - 目的：描述固定位置的相对调制深度。
4. `GZONE{1,2,3}_PERIODICITY`
   - 定义：40–180 bpm 延迟范围内的最大归一化自相关。
   - 目的：判断局部弱信号是否仍有可信生理周期。
5. `GZONE{1,2,3}_AMB_ABS_CORR`
   - 定义：`abs(corr(zone_pulse, ambient_pulse))`。
   - 目的：识别固定位置的环境光耦合和漏光污染。

新增候选统一归入 `green_position` 组，并带 `fixed_position`、`device_shortcut`、`cross_mode_generalization` 风险标记。相对对比度和相关性声明有界范围；AC/DC 使用既有安全除法和数值保护。

## 共享模型约束

- 三种硬件配置必须先归一到同样的三个固定区域，再计算位置特征。
- 第一版不增加原始通道数量和区域内部通道离散度，避免模型直接识别硬件配置。
- `mode` 仍只是普通候选；位置特征与 mode 均需审计跨 subject/device/session/mode 泛化。
- 特征池版本升级，旧排序、旧人工 CSV 和旧模型 bundle 不允许静默复用。

## 验收

- 目录和窗口提取顺序完全一致，候选总数由 111 增至 126。
- 15 个位置候选在正常、常量、极小量和单区异常输入上均为有限值。
- 交换区域信号时，位置特征随物理区域交换；原有置换不变特征保持不变。
- mode1 的三区域输入与 mode2 的九路绿光按区平均后得到一致的位置特征。
- 所有新增候选具备公式、单位、C 操作符、成本、风险和有界范围元数据。
- 人工 CSV、部署公式、C contract 和 golden vectors 能按最终选择顺序包含任意新增候选。
- README 明确说明固定位置语义、共享模型和人工选择方式。
