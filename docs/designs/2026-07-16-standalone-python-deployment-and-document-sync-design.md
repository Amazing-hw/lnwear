## 1. 目标

最终部署只需要两个项目产物：

1. `deploy_feature_extractor.py`：独立完成窗口特征提取、训练侧填充/裁剪、特征顺序固化和窗口阈值判断。
2. `final_model.json`：与上述特征顺序完全一致的 XGBoost 模型。

部署目录不需要项目源码、`s03_extract_feature_pool.py`、特征目录 JSON、cookbook、
训练 bundle 或原始 artifact 目录。Python 运行环境需要 NumPy、SciPy 和 XGBoost；这些是
第三方运行时依赖，不属于项目源码依赖。

## 3. 独立部署脚本

### 3.1 生成方式

`s08_run_pipeline.py` 在导出时从当前受治理的 `s03` 实现抽取特征计算引擎，并将其源码
内联到 `deploy_feature_extractor.py`。生成文件同时内嵌：

- `FEATURE_ORDER`；
- 每个入选特征的 `FILL_VALUES`；
- 每个入选特征的 `CLIP_BOUNDS`；
- `FEATURE_FORMULAS`；
- `WINDOW_MODEL_THRESHOLD`；
- 默认采样率和窗口长度；
- 与训练一致的三光区、环境光和 ACC 预处理及特征计算函数。

生成文件不得包含 `S03_SOURCE_DIR`、`sys.path` 修改或对任意项目 Python 模块的导入。
唯一允许的导入是 Python 标准库及明确的第三方运行时库。

### 3.2 部署接口

保留现有接口：

- `extract_raw_feature_dict(...)`：输出按人工选择冻结后的原始特征字典；
- `extract_feature_dict(...)`：应用训练侧填充和裁剪；
- `extract_features(...)`：输出与模型输入顺序一致的数值向量；
- `classify_probability(...)`：按训练固化阈值将概率转为单窗标签。

调用方使用 `extract_features(...)` 得到二维模型输入，再用 XGBoost 加载
`final_model.json` 进行概率预测。部署脚本不复制模型文件，也不静默寻找其他 artifact。

### 3.3 失败策略

- 选中特征未由内联引擎产生时立即抛出 `KeyError`，不得填入一个未定义的新特征。
- 输入为空时沿用现有特征缺失处理契约。
- 非有限值按训练 bundle 固化的 fill/clip 规则处理。
- 导出时若模型包含 Stage2 IR 或不可部署特征，继续拒绝生成产物。

## 4. 测试设计

先增加会在当前实现上失败的回归测试：

1. 生成脚本不包含 `S03_SOURCE_DIR`、`sys.path.insert` 或
   `from s03_extract_feature_pool import ...`。
2. 将脚本复制到与项目源码无关的临时目录，清空项目路径后仍可编译、导入并完成特征提取。
3. 对全部 126 个受治理候选，独立脚本与训练侧 `s03` 在确定性窗口上的结果逐项一致。
4. 对人工选择的子集，输出名称、顺序、填充、裁剪和阈值与模型 bundle 一致。
5. 端到端守卫明确只要求 `deploy_feature_extractor.py` 和 `final_model.json` 即可执行
   Python 单窗推理；其他 JSON 是审计/固件移植辅助文件，不是运行时必需文件。

测试必须先在旧实现上因源码依赖失败，再接入内联引擎并转绿。

## 6. 每条数据的首尾窗口裁剪

保持实现简单：每条数据读入后先按窗口编号稳定排序，然后直接使用 `[3:-3]`，使 PPG、
ACC、窗口编号和窗口标签同步删除最前 3 包与最后 3 包。后续特征提取、训练、评估和缓存
只接收裁剪后的窗口，不新增尾部裁剪参数或缓存版本。

连续时序输入在生成完整有序窗口起点列表后同样直接使用 `[3:-3]`。若总窗口数不超过 6，
裁剪结果为空并沿用现有“无合法 Stage2 窗口”回退和统计逻辑，不静默保留边界窗口。
现有 `skip_initial_windows` 仅作为默认值为 0 的额外兼容选项，不参与自动首尾三包裁剪。
